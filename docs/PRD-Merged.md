## 一、产品概述

| 项目 | 内容 |
| --- | --- |
| 产品名称 | **PetDesk** — 桌面宠物搭子 |
| 一句话定位 | 用**真实视频质感**的桌面宠物，提供**窗口级深度互动** + 轻量效率辅助 |
| 核心差异化 | **视频级真实视觉效果** × **跨窗口感知联动** × **鼠标/环境深度互动**。传统 2D 桌宠动作生硬，我们的视频素材 + 窗口感知让宠物真正"活"在桌面上 |
| 目标用户 | 久坐上班族、程序员、学生、二次元爱好者（22-35 岁，日均用电脑 ≥6h） |
| 技术栈 | Electron + 透明视频播放 (WebM Alpha) + Canvas 叠加层 |
| 核心目标 | 2 周 MVP 验证视觉吸引力，后续迭代完善深度互动，实现被动收入 |

### 为什么选择视频渲染而非 2D 精灵图？

| 维度 | 2D 精灵图（传统桌宠） | 视频素材（本产品） |
| --- | --- | --- |
| 视觉效果 | 帧动画、动作生硬、质感差 | 真实宠物/高质量 3D 渲染、动作流畅 |
| 素材制作 | 需要逐帧手绘或骨骼动画，成本高 | 录制真实宠物视频抠绿幕 / AI 生成 / 3D 渲染导出 |
| 新鲜感周期 | 3-7 天便觉单调 | 视频素材本身具备丰富的微表情和随机动作 |
| 可定制性 | 用户几乎无法自制 | 用户可用自家宠物视频，剪映/ CapCut 抠背景后直接导入 |
| 传播属性 | 一般 | 短视频截取效果极佳，天然适合 B 站/小红书传播 |

## 四、渲染引擎设计（核心变更）

### 4.0 技术方案：透明视频 + Canvas 叠加

```
┌──────────────────────────────┐
│   Electron 无边框透明窗口     │
│  ┌────────────────────────┐  │
│  │  <video> 透明宠物视频    │  │  ← 视频层：播放 WebM Alpha 视频
│  │  (WebM VP9 Alpha)       │  │
│  └────────────────────────┘  │
│  ┌────────────────────────┐  │
│  │  <canvas> 叠加层         │  │  ← Canvas 层：粒子特效、进度条、
│  │  (天气/气泡/进度条)      │  │     节日头饰、文字气泡
│  └────────────────────────┘  │
└──────────────────────────────┘
```

**视频格式方案**：

| 方案 | 透明度支持 | Electron/Chromium 兼容性 | 建议 |
| --- | --- | --- | --- |
| **WebM VP8/VP9 Alpha** | ✅ 原生 Alpha 通道 | ✅ Chromium 全支持 | **首选** |
| HEVC Alpha (MP4) | ✅ | ❌ 仅 Safari 支持 | 备用（macOS 用户） |
| 双视频合成（颜色+遮罩） | ✅ 通过 Canvas 合成 | ✅ 全平台 | 备选方案 |

**视频素材规格**：
- 格式：WebM（VP9 编码 + Alpha 通道）
- 分辨率：≤1080p（默认 500×500 以内）
- 帧率：30fps
- 单段时长：2-10 秒（循环播放）
- 单文件大小：≤5MB（单个动作素材）
- 制作方式：录真宠抠绿幕 / AI 视频生成工具 / 3D 软件渲染

### 4.1 视频素材管理

**素材结构**：

```json
{
  "pets": [
    {
      "id": "default-cat",
      "name": "小橘",
      "preview": "assets/preview/cat-thumb.png",
      "actions": {
        "idle":          { "src": "assets/cat/idle.webm",      "loop": true,  "duration": 6 },
        "walk":          { "src": "assets/cat/walk.webm",      "loop": true,  "duration": 4 },
        "happy":         { "src": "assets/cat/happy.webm",     "loop": false, "duration": 3 },
        "surprised":     { "src": "assets/cat/surprised.webm", "loop": false, "duration": 2 },
        "sleep":         { "src": "assets/cat/sleep.webm",     "loop": true,  "duration": 8 },
        "scared":        { "src": "assets/cat/scared.webm",    "loop": false, "duration": 2 },
        "celebrate":     { "src": "assets/cat/celebrate.webm", "loop": false, "duration": 4 },
        "remindDrink":   { "src": "assets/cat/drink.webm",     "loop": false, "duration": 3 },
        "remindRest":    { "src": "assets/cat/rest.webm",      "loop": false, "duration": 3 },
        "warnBoss":      { "src": "assets/cat/warn.webm",      "loop": true,  "duration": 3 }
      }
    }
  ]
}
```

**视频切换流程**：
1. 用户操作 / 系统事件 → 触发动作名称
2. VideoController 查找对应素材 → 预加载（如果未缓存）
3. 当前视频 crossfade（200ms 透明度过渡） → 新视频播放
4. `loop: true` → 循环播放直到下次切换；`loop: false` → 播放完毕后自动回 `idle`

## 五、核心互动系统

### 5.1 跨窗口位置联动 ★ 核心卖点

> 宠物不局限于桌面悬浮，实时跟随前台窗口位置，产生"陪伴你在工作"的真实感。

| 场景 | 宠物行为 | 视频素材 |
| --- | --- | --- |
| 窗口最大化 | 蹲坐在窗口顶部边缘外侧，随机晃脚、低头看屏幕 | idle（循环）+ 随机插入 lookDown |
| 窗口缩小 / 移动 | 沿窗口边缘滑动，保持贴在边框上 | walk（循环）+ 窗口坐标实时 tween |
| 切换活动窗口 (Alt+Tab) | 从旧窗口跳下 → 跑动 → 爬上新高亮窗口 | jumpDown → walk → climbUp（过渡序列） |
| 窗口最小化 | 宠物回到桌面自由活动 | idle / walk / sleep 随机 |

**技术实现**：
- 主进程每 150ms 轮询前台窗口坐标（`active-win`）→ IPC 推送渲染进程
- 渲染进程 tween 窗口位置（ease-out, 200ms），掩盖轮询间隔
- 窗口切换时先触发过渡视频，到达目标位置后切回 idle

### 5.2 桌面环境感知

| 检测场景 | 判定方式 | 宠物反应 |
| --- | --- | --- |
| IDE / 代码编辑器 | 窗口标题匹配（VS Code, IntelliJ, Vim 等） | 蹲在窗口侧边盯屏，偶尔伸爪子 |
| 浏览器 | 进程名匹配 | 趴在底部边缘打滚 |
| 视频网站 | 标题匹配（YouTube, Bilibili 等） | 凑到画面旁微微晃头 |
| 空白桌面 | 无前台窗口匹配 | 自由走动、打滚、睡觉 |
| 剪贴板大段复制 | 文本长度 >200 字符 | 惊讶跳起 → 歪头 |

### 5.3 鼠标深度互动（7 种）

| 用户操作 | 宠物反应 | 冷却 |
| --- | --- | --- |
| 悬停宠物 ≥1s | 抬头注视，耳朵/眼睛跟随光标 | 无 |
| 单击宠物 | 播放"被抚摸"视频 + 弹出问候语气泡 + 心情 +2 | 0.5s |
| 快速拖动经过（>2000px/s） | 受惊躲到屏幕角落，5s 后探头回来 | 10s |
| 鼠标静置 ≥3min | 跑来扒拉光标，播放吸引注意视频 | 5min |
| 双击宠物 | 随机特殊动作（后空翻 / 打喷嚏） | 2s |
| 右键宠物 | 弹出上下文菜单（表情包/设置/切换素材） | 无 |
| 拖动文件经过 | 追逐鼠标轨迹 | 3s |

### 5.4 桌面拖动与吸附

- 用户可拖拽宠物到任意位置
- 拖到屏幕边缘 ±20px → 自动吸附贴边
- 贴边状态下宠物可部分超出屏幕（露出半个身体）
- 拖动时暂停窗口跟踪（用户手动定位优先），30s 无拖动后恢复跟踪

## 六、情绪养成系统

### 6.1 场景化情绪反馈

| 触发条件 | 宠物行为 |
| --- | --- |
| 连续输入 ≥2h | 播放"叼水杯"视频 + 气泡："歇会儿呀，喝口水~" |
| 23:00-06:00 | 播放"犯困"视频（哈欠、半睁眼）、移速 ×0.3、不主动互动 |
| 周五 18:00+ | 播放"庆祝"视频（转圈蹦跶） |
| 30min 无互动 | 垂头丧气趴在角落 |
| 系统唤醒 | 从屏幕外跑进来迎接 |

> 约束：同类型气泡每小时最多 1 次，显示 ≤8s，不遮挡操作区域。

### 6.2 性格动态塑造（3 参数）

```json
{
  "intimacy": 0,    // 亲密度：抚摸 ↑ → >70 主动靠近鼠标 +50%
  "energy": 50,     // 活跃度：专注模式 ↓ 娱乐 ↑ → <30 少跑动
  "tsundere": 0     // 傲娇值：不理睬 ↑ → >60 点击时 30% 概率扭头躲开
}
```

### 6.3 仪式感彩蛋

| 类型 | 触发 | 表现 |
| --- | --- | --- |
| 天气联动 | OpenWeather API | Canvas 叠加雨滴/雪花粒子 |
| 节日 | 系统日期 | Canvas 叠加头饰（圣诞帽/红包/恶魔角） |
| 纪念日 | 用户自定义 | 播放"蛋糕"视频 + Canvas 烟花粒子 |
| 随机稀有动作 | 启动时 5% | 播放特殊视频（倒立/太空步） |

## 七、效率辅助系统（Pro 核心）

### 7.1 隐蔽式摸鱼预警 ★ 核心付费点

| 功能 | 描述 |
| --- | --- |
| 自定义规则 | 关键词 / 进程名匹配（企业微信、钉钉、OA 系统） |
| 预警动作 | 宠物快速跑向角落 + 播放"拍爪子警告"视频 + 可选提示音 |
| 紧急隐藏 | 双击宠物 → 隐藏到系统托盘，双击托盘恢复 |
| 误报处理 | 气泡"忽略"按钮 → 该规则权重降低 |

### 7.2 休息 & 饮水提醒

| 功能 | 描述 |
| --- | --- |
| 休息提醒 | 默认间隔 40min，可自定义，到时间播放"提醒休息"视频 + 气泡 |
| 饮水提醒 | 默认间隔 60min，独立开关，到时间播放"提醒喝水"视频 + 气泡 |
| 关闭 | 提醒可手动关闭，关闭后宠物恢复正常 |

### 7.3 待办进度可视化

- 用户添加待办 + 目标量 → 宠物身旁 Canvas 进度条
- 0%：垂头丧气视频 → 50%：精神振作 → 100%：庆祝视频 + Canvas 彩带粒子
- 连续 3 天 100%：解锁新动作视频；连续 7 天：解锁新装扮

### 7.4 订阅账单提醒

- 手动录入（名称、日期、金额、周期）
- 到期前 3 天：播放"叼提醒条"视频
- 扣费当天：播放"举账单"视频
- 月底汇总气泡

## 八、衍生玩法

### 8.1 表情包/GIF 生成

- 右键 →「生成表情包」或 Ctrl+Shift+E
- 截取当前视频帧 + 去除透明背景 → 叠加文字
- 内置 20+ 文案模板，支持自定义
- 一键复制到剪贴板，可直接粘贴到微信/钉钉/飞书
- **视频专属优势**：可导出 2-3 秒循环 GIF 动态表情

### 8.2 好友宠物串门（实验性，V2.0+）

- P2P WebRTC 直连，好友码配对
- 对方宠物随机串门，播放"串门"视频，停留 5-10min
- 可带 ≤50 字留言气泡

## 九、素材管理系统（差异化功能）

### 9.1 用户上传自有素材

- 上传 WebM / MP4 视频，支持带透明通道或绿幕背景
- 内置简易抠色工具（Canvas chroma-key，设定要移除的背景色）
- 上传后自动生成预览缩略图
- 存储于本地，不上传服务器

### 9.2 素材绑定互动

```
┌─────────────────────────────────┐
│  素材: mycat_happy.webm          │
│  绑定到: [点击互动] [休息提醒]    │
│  问候语: "喵~今天也很开心"        │
│  [保存绑定] [恢复默认]           │
└─────────────────────────────────┘
```

- 每个用户素材可选择绑定到任一互动事件
- 绑定后触发互动时优先播放用户素材
- 支持解绑恢复默认

### 9.3 素材商店（后续迭代）

- 提供官方制作的精品宠物视频素材包
- 按宠物品种/风格分类：猫咪、柴犬、柯基、二次元
- 定价 9.9 元/包，Pro 用户享折扣

## 十、非功能需求

| 维度 | 指标 | 说明 |
| --- | --- | --- |
| **内存** | 闲置 ≤200MB，播放中 ≤300MB | 视频解码占用高于纯 Canvas，放宽 50MB |
| **CPU** | 闲置 ≤8%，播放 ≤15% | 视频硬件解码 (GPU) 为主，CPU 开销低 |
| **启动速度** | 冷启动 ≤4s | 含首段视频预加载 |
| **视频帧率** | 30fps 稳定播放 | 依赖 GPU 硬件解码 |
| **平台** | Win 10/11 优先，macOS 11+ 次要 | Chromium 视频解码统一 |
| **离线** | 核心功能全离线；天气/串门需联网可选关闭 | |
| **隐私** | 仅读窗口标题+位置；不采集输入/文件/剪贴板内容；用户视频存本地 | |
| **视频格式** | 输入：WebM (VP9 Alpha) 首选，MP4 (HEVC Alpha) 备选 | |

## 十三、技术可行性评估

### 13.1 风险矩阵

| 功能 | 难度 | 风险 | 说明 |
| --- | --- | --- | --- |
| 透明视频播放 (WebM Alpha) | ⭐⭐ | 🟡 中 | Chromium 支持 VP8/VP9 Alpha，但需验证 Electron 版本兼容；备选方案：Canvas chroma-key 合成 |
| 跨窗口位置联动 | ⭐⭐⭐⭐ | 🔴 高 | 同之前评估，多 OS/分辨率适配是最大难点 |
| 视频切换流畅性 | ⭐⭐ | 🟡 中 | 需预加载 + crossfade，大视频文件切换可能有延迟 |
| 鼠标互动 | ⭐⭐ | 🟢 低 | 成熟方案 |
| 环境感知 | ⭐⭐ | 🟡 中 | 窗口标题匹配可控 |
| Canvas 叠加层 | ⭐ | 🟢 低 | 成熟前端技术 |
| 用户视频上传处理 | ⭐⭐⭐ | 🟡 中 | chroma-key 抠色效果取决于视频质量；透明通道视频直接支持 |
| 好友串门 | ⭐⭐⭐⭐ | 🔴 高 | 非 MVP 范围 |

### 13.2 视频格式兼容性详情

```
WebM (VP9 + Alpha)  →  Chromium 80+  ✅  →  Electron 28+ ✅
HEVC Alpha (MP4)    →  仅 Safari      ❌  →  Electron 不可用
双视频合成           →  全平台         ✅  →  实现复杂但兜底
Canvas Chroma-key   →  全平台         ✅  →  最简单兜底方案
```

**推荐 MVP 方案**：WebM VP9 Alpha 为首选；对于用户上传的不带透明通道的视频，提供简易 Canvas chroma-key 工具。

## 十四、风险与应对

| 风险 | 应对 |
| --- | --- |
| WebM Alpha 在某些 Electron 版本不兼容 | 降级方案：Canvas chroma-key（绿幕抠色），牺牲边缘精度但保证可用 |
| 视频素材文件大，切换卡顿 | 预加载全部动作视频到内存；低性能模式下降低视频分辨率 |
| 用户上传视频格式多样 | 内置 FFmpeg 自动转码为 WebM Alpha（打包时包含 ffmpeg 二进制） |
| 窗口坐标偏差 | 提供用户手动校准 + 社区共享分辨率适配配置 |
| 新鲜感流失 | 视频素材本身比 2D 耐看；养成系统 + 彩蛋 + 定期更新素材包 |
| 冷启动无流量 | 视频质感在短视频平台传播力极强，录制效果视频发布 |

## 附录：与原 PawDesk PRD 的核心变更

| 维度 | 原 PawDesk PRD | 合并后 |
| --- | --- | --- |
| **渲染方式** | Canvas 精灵图序列帧 | `<video>` 播放 WebM Alpha 透明视频 |
| **素材制作** | 手绘/骨骼动画（需美术） | 录制真宠 / AI 生成 / 3D 渲染后抠背景 |
| **用户定制** | 不可自定义 | 可上传自有视频 + 简易抠色工具 |
| **视觉效果** | 2D 卡通/像素风 | 真实宠物质感或高质量 3D 渲染 |
| **美术成本** | 需要画师/动画师 | 可用 AI 工具 + 录制素材替代 |
| **视频切换** | 动画状态机控制精灵图帧 | 视频源切换 + crossfade |
| **Canvas 角色** | 主渲染引擎 | 降级为叠加层（粒子/UI/进度条/头饰） |
| **互动系统** | 不变 | 完全保留 + 新增用户视频绑定互动 |
| **效率功能** | 不变 | 完全保留 + 新增休息/饮水提醒 |
| **技术栈** | Electron + Canvas | Electron + `<video>` + Canvas 叠加 |
| **视频格式** | — | WebM VP9 Alpha (首选) + chroma-key 兜底 |
