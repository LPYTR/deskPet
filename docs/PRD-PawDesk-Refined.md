## 一、文档概述

| 项目 | 内容 |
| --- | --- |
| 产品名称 | PawDesk 桌面搭子 |
| 产品定位 | 桌面端场景化互动宠物软件，主打「治愈陪伴 + 轻量效率」，通过窗口感知、行为联动打造真实陪伴感，区别于传统静态 GIF 桌宠 |
| 目标用户 | **核心**：久坐上班族、程序员、新媒体运营（22-35 岁，日均使用电脑 ≥6h）；**次要**：学生群体、二次元爱好者 |
| 技术栈 | **推荐 Tauri (Rust) + 前端动效（Canvas / Lottie / Spine）**；备选 Electron + Canvas。前端独立可完成全栈开发 |
| 核心目标 | 打造差异化桌面宠物产品，MVP 验证用户付费意愿，实现被动收入 |

> **技术栈说明**：原 PRD 建议 Electron / Tauri 二选一。经评估，Tauri 在性能（内存 <50MB 基础开销 vs Electron ~150MB）、窗口 API 调用能力上更适合本产品。对不熟悉 Rust 的开发者，可先用 Electron 快速出 MVP，后续迁移到 Tauri。

## 二、用户痛点与产品价值

### 2.1 用户核心痛点

| 痛点 | 现状 | 机会 |
| --- | --- | --- |
| 传统桌宠仅为悬浮 GIF，行为固定无交互 | 3-7 天新鲜感流失即卸载 | 窗口互动 + 鼠标互动形成真实陪伴感 |
| 上班族工作枯燥，需要低成本解压 | 不想打开复杂的游戏/App，但需要情绪出口 | 随手可及的轻互动，零学习成本 |
| 待办/专注类工具过于冰冷 | 缺乏正向反馈，难以坚持使用 | 宠物化包装让效率工具变"软" |
| 订阅会员多，常忘扣费日期 | 被动支出增加，年轻人普遍痛点 | 宠物提醒将痛点转化为功能记忆点 |

### 2.2 产品核心价值

- **情绪价值**：鲜活的场景化互动 → 轻量治愈的工位陪伴感
- **效率价值**：宠物化效率工具 → 降低使用门槛，提升执行动力
- **差异化壁垒**：跨窗口互动 + 环境感知两大核心技术点，竞品短期难以复制；极易通过短视频出圈传播
- **隐私优先**：核心功能全部本地运行，不采集用户输入内容，建立用户信任

## 四、详细功能需求

### 4.1 核心互动系统（MVP 必做）

#### 4.1.1 跨窗口位置联动 ★ 核心卖点

> **功能描述**：宠物不局限于桌面顶层悬浮，可与用户当前活动窗口产生位置联动，打破悬浮窗的割裂感。

**交互逻辑**：

| 场景 | 宠物行为 | 优先级 |
| --- | --- | --- |
| 窗口最大化 | 宠物蹲坐在窗口**顶部边缘外侧**，随机触发晃脚、低头看屏幕、甩尾巴动作 | P0 |
| 窗口缩小/移动 | 宠物沿窗口边缘滑动，保持趴在边框上的状态 | P0 |
| 切换活动窗口 | 宠物从旧窗口边缘跳下 → 跑动动画 → 爬上新高亮窗口 | P0 |
| 鼠标拖拽窗口边缘调整大小时 | 宠物被"挤到"一边，做出受惊缩头的反应 | P1 |

**技术实现要点**：
- 通过系统 API 获取前台窗口句柄与屏幕坐标（Windows: `GetForegroundWindow` + `GetWindowRect`；macOS: `NSWorkspace` + Accessibility API）
- 将宠物窗口定位为 `alwaysOnTop` + `transparent` 无边框窗口，根据目标窗口坐标实时计算贴边位置
- 窗口切换时播放过渡动画（跑动/跳跃），动画时长 300-600ms，遮盖位置计算的微小延迟
- **边界情况**：多显示器环境需获取虚拟桌面总坐标；窗口跨屏拖动时宠物自动跟随

> **⚠️ 现实考量**：此功能为本产品最大技术难点。不同 OS / 窗口管理器下获取精确窗口坐标存在差异，MVP 阶段优先适配主流分辨率（1920×1080、2560×1440），并预留用户手动微调宠物的校准入口。macOS 需申请辅助功能权限。

#### 4.1.2 桌面环境感知互动

> **功能描述**：宠物根据用户当前前台应用类型，自动切换行为模式。

**交互逻辑**：

| 检测到的场景 | 判定方式 | 宠物行为 |
| --- | --- | --- |
| IDE / 代码编辑器 | 窗口标题匹配关键词（VS Code, IntelliJ, Sublime, Vim, NeoVim 等） | 蹲在窗口侧边"盯屏"，偶尔伸爪子碰光标位置 |
| 浏览器 | 进程名匹配（chrome.exe, firefox, Safari 等） | 趴在窗口底部边缘，偶尔打滚 |
| 视频/娱乐（可进一步细分） | 窗口标题匹配（YouTube, Bilibili, Netflix, 抖音 等） | 凑到画面旁，随播放进度微微晃头 |
| 空白桌面（无前台窗口） | 无匹配的活跃窗口 | 宠物在桌面自由走动、打滚、睡觉 |
| 剪贴板大段文本复制 | 监听剪贴板文本长度（>200 字符） | 宠物惊讶跳起 → 歪头 → 恢复 |

**技术实现要点**：
- 前台应用识别仅依赖**窗口标题 + 进程名**的规则匹配（用户可自定义规则），不读取窗口内容
- 提供一个用户可编辑的 `app-rules.json` 配置文件，社区可贡献规则
- 剪贴板监听仅获取文本长度，不读取实际内容，确保隐私安全

> **⚠️ 现实考量**：原 PRD 中"图标密集区域绕路行走"依赖于分析桌面图标布局，实现成本高且不同 OS 差异大。MVP 阶段改为：宠物默认活动区域为桌面**非密集区域**（屏幕下方/角落），支持用户拖拽设定宠物的"活动范围"。

#### 4.1.3 鼠标深度互动

> **功能描述**：基于鼠标操作的高频轻互动，零学习成本，随手可触发。

| 用户操作 | 宠物反应 | 冷却时间 |
| --- | --- | --- |
| 鼠标悬停在宠物上 ≥1s | 抬头注视光标，耳朵/眼睛跟随光标移动 | 无 |
| 左键单击宠物 | 抚摸动画，宠物开心反应（蹭手/翻滚），心情值 +2 | 0.5s |
| 鼠标快速拖动经过宠物（速度 >阈值） | 受惊炸毛，躲到屏幕最近角落，5s 后探头回来 | 10s |
| 鼠标静置 ≥3 分钟 | 宠物跑过来用爪子扒拉光标吸引注意，发出轻互动音效 | 5min |
| 拖动文件/图标时 | 宠物追着鼠标轨迹跑动，试图"拦截" | 3s |
| 双击宠物 | 触发特殊动作（后空翻/打喷嚏），随机播放 | 2s |
| 右键宠物 | 弹出上下文菜单（表情包/设置/喂食） | 无 |

> **⚠️ 原 PRD 调整**："鼠标快速拖动"判定需要在短时间内追踪鼠标移动速度，存在性能开销。建议以 50ms 采样间隔计算鼠标瞬时速度，超过 2000px/s 触发。

### 4.2 情绪养成系统

#### 4.2.1 场景化情绪反馈

> **功能描述**：基于使用时长、时段、操作频率自动触发情绪反馈，主动提供陪伴价值。

| 触发条件 | 宠物行为 |
| --- | --- |
| 连续键盘输入 ≥2 小时 | 叼着水杯图标走到屏幕中央，气泡："歇会儿呀，喝口水~" |
| 夜间 23:00-06:00 持续使用 | 宠物打哈欠、眼睛半睁、行动缓慢（移速降至 30%），不主动发起互动 |
| 周五 18:00 后 | 原地转圈蹦跶，触发小庆祝动画 |
| 长时间未互动（≥30min 活跃但无鼠标接触） | 宠物垂头丧气趴在角落，心情图标下降 |
| 用户刚解锁电脑（系统唤醒事件） | 宠物从屏幕外跑进来，开心迎接 |

> **设计约束**：气泡提示每次显示不超过 8s，不强制遮挡用户操作区域。同一类型提示每小时最多触发 1 次，避免打扰。

#### 4.2.2 性格动态塑造（轻量版）

> **功能描述**：用户的互动习惯逐步改变宠物性格参数。不追求复杂的行为树，而是通过少数几个可量化参数驱动差异化体验。

**性格维度（MVP 阶段 3 个参数）**：

| 参数 | 范围 | 影响因素 | 行为变化 |
| --- | --- | --- | --- |
| 亲密度 | 0-100 | 抚摸次数 ↑、互动频率 ↑ | 亲密度 >70：主动靠近鼠标概率 +50% |
| 活跃度 | 0-100 | 专注模式开启 ↑ → 活跃度 ↓；娱乐时间 ↑ → 活跃度 ↑ | 活跃度 <30：跑动减少，更多趴在角落 |
| 傲娇值 | 0-100 | 长期不互动 ↑ | 傲娇值 >60：点击时 30% 概率扭头躲开 |

> **⚠️ 现实考量**：原 PRD 中"性格动态塑造"若完整实现需要复杂的行为树系统，MVP 阶段建议仅保留 3 个可量化参数，在用户本地存储为 JSON。更深度的性格系统可在 V1.2+ 迭代。

#### 4.2.3 随机仪式感彩蛋

| 彩蛋类型 | 触发条件 | 表现 |
| --- | --- | --- |
| 天气联动 | 调用 OpenWeather API 获取本地天气 | 雨天：身上水珠效果 / 雪天：积雪效果 |
| 节日彩蛋 | 系统日期匹配预设节日表 | 圣诞帽 / 春节红包 / 万圣节小恶魔角 |
| 用户自定义纪念日 | 用户设置日期 | 宠物推出小蛋糕 + 烟花粒子动画 |
| 随机彩蛋（低概率） | 每次启动时 5% 概率触发 | 宠物做出随机稀有动作（倒立/太空步等） |

> **⚠️ 实现建议**：天气联动需联网，作为可选功能（用户可关闭）。节日头饰通过图层叠加实现，降低美术资源成本。项目已有的 `OPENWEATHER_API_KEY` 环境变量可直接复用。

### 4.3 效率辅助系统（Pro 版核心付费功能）

#### 4.3.1 隐蔽式摸鱼预警 ★ 核心付费点

> **功能描述**：用宠物动作替代生硬的老板键，职场场景刚需。

| 功能点 | 描述 |
| --- | --- |
| 自定义预警规则 | 用户添加关键词（如"老板"、"OA"）或进程名（如企业微信、钉钉），当匹配窗口出现在前台时触发 |
| 预警动作 | 宠物快速跑到屏幕边角，连续拍爪子；同时可选播放轻提示音 |
| 紧急隐藏 | 双击宠物 → 立即隐藏至系统托盘，恢复时再次点击托盘图标 |
| 误报忽略 | 用户可在预警气泡上点"忽略此类提醒"，规则自动降低权重 |

> **⚠️ 原 PRD 调整**：原方案中"缩小为 1px 圆点隐藏在任务栏缝隙"在跨平台实现上极其困难且不稳定。改为**最小化到系统托盘**，双击托盘图标恢复，功能等价但实现可靠。系统托盘在 Windows/macOS 均有标准 API。

#### 4.3.2 待办进度可视化养成

| 功能点 | 描述 |
| --- | --- |
| 添加待办 | 用户输入任务 + 目标量（代码行数/文档字数/番茄钟数），支持简单分类 |
| 进度条展示 | 宠物头顶/身旁显示浮动的迷你进度条 |
| 里程碑反馈 | 0%：垂头丧气 → 50%：精神振作、眼睛发光 → 100%：庆祝动画（彩带粒子 + 欢呼跳跃） |
| 连续完成奖励 | 连续 3 天 100% → 解锁限定动作；连续 7 天 → 解锁限定装扮 |
| 数据本地存储 | 所有待办和进度存储在本地 SQLite / JSON，无需云同步 |

#### 4.3.3 订阅账单到期提醒

| 功能点 | 描述 |
| --- | --- |
| 添加订阅 | 用户手动录入：名称、扣费日期、金额、周期（月/季/年） |
| 到期提醒 | 扣费前 3 天：宠物叼着提醒条跑到屏幕中央；扣费当天：宠物举账单跳舞 |
| 月度汇总 | 每月最后一天，宠物展示月度订阅总支出气泡，点击可查看明细 |
| 隐私保证 | 订阅数据仅本地存储，不采集不上传 |

### 4.4 衍生玩法系统

#### 4.4.1 好友宠物串门（实验性功能，V2.0+）

> **功能描述**：添加好友后，双方在线时宠物可互相串门。

| 功能点 | 描述 |
| --- | --- |
| 添加好友 | 通过好友码配对，P2P 直连（WebRTC），无需中心服务器 |
| 串门行为 | 对方宠物随机串门，携带小零食动画，停留 5-10 分钟后自动返回 |
| 留言功能 | 串门时可带一句留言（≤50 字），由宠物气泡展示 |
| 隐私控制 | 可设置"请勿打扰"模式，拒绝串门请求 |

> **⚠️ 现实考量**：此功能需要 P2P 网络（WebRTC + 信令服务器），引入后端基础设施和 NAT 穿透等复杂问题。**建议标记为实验性功能**，仅列入 V2.0 长期规划。MVP 阶段聚焦单机体验。

#### 4.4.2 一键生成表情包

| 功能点 | 描述 |
| --- | --- |
| 触发方式 | 右键宠物 →「生成表情包」；或快捷键 Ctrl+Shift+E |
| 截图范围 | 自动裁剪宠物当前画面，去除透明背景 |
| 文案模板 | 内置 20+ 常用模板（"摸鱼中"、"已宕机"、"划水中"），支持自定义文字 |
| 分享方式 | 一键复制到剪贴板 → 直接粘贴到微信/钉钉/飞书 |

### 4.5 基础设置系统

| 模块 | 功能 |
| --- | --- |
| 宠物形象 | 默认 1 款基础形象，Pro 版解锁更多；支持后续购买主题包 |
| 性能档位 | 低档（30fps，集显友好）/ 中档（60fps）/ 高档（120fps，独立显卡），默认中档 |
| 显示设置 | 开机自启、音量（0-100%）、宠物大小缩放（0.5x-2x）、显示层级 |
| 活动区域 | 用户可拖拽设定宠物的主要活动范围，避免遮挡常用区域 |
| 数据管理 | 所有配置和养成数据本地 SQLite/JSON 存储，支持导出/导入备份 |

> **⚠️ 新增**：性能档位对低配电脑用户至关重要。Canvas 动画在低档模式下减少帧率和粒子特效，确保闲置 CPU ≤5%。

## 五、非功能需求

| 维度 | 指标 |
| --- | --- |
| **内存** | 闲置状态 ≤150MB（Electron）或 ≤80MB（Tauri）；动画运行峰值 ≤200MB |
| **CPU** | 闲置 ≤5%；60fps 动画运行时 ≤10%；30fps 模式下 ≤5% |
| **启动速度** | 冷启动 ≤3s |
| **平台** | Windows 10/11（首要）、macOS 11+（次要）；Linux 列入长期规划 |
| **离线** | 核心功能（互动、效率、养成）全部本地运行；天气/串门需联网但可选关闭 |
| **隐私** | 仅读取窗口标题与位置；不采集键盘输入内容、文件内容、剪贴板内容；不上传任何用户数据 |
| **无障碍** | 支持纯键盘触发紧急隐藏；颜色对比度满足 WCAG AA 标准 |

## 六、变现模式设计

采用 **「免费基础版 + 一次性买断 Pro 版」**模式，降低决策门槛：

| 版本 | 权益 | 价格 |
| --- | --- | --- |
| **免费版** | 1 款基础形象 + 核心互动系统 + 基础情绪反馈 + 1 个待办 + 3 个订阅监控位 | 免费 |
| **Pro 版** | 全部宠物形象 + 全部效率功能 + 性格塑造 + 节日彩蛋 + 表情包生成 + 无限待办/订阅 | **29 元** 一次性买断 |
| **主题形象包** | 猫咪系列 / 柴犬系列 / 二次元系列 / 节日限定（后续推出） | **9.9 元/个** |

## 八、技术可行性评估

### 8.1 关键风险点评级

| 功能 | 技术难度 | 风险等级 | 说明 |
| --- | --- | --- | --- |
| 跨窗口位置联动 | ⭐⭐⭐⭐ | 🔴 高 | 多 OS 窗口 API 差异大，需逐平台适配；多显示器/缩放比场景复杂 |
| 桌面环境感知 | ⭐⭐ | 🟡 中 | 窗口标题匹配简单可靠；进程名获取跨平台有差异但可控 |
| 鼠标深度互动 | ⭐⭐ | 🟢 低 | 前端技术成熟，全局鼠标钩子有成熟方案 |
| 情绪/性格系统 | ⭐ | 🟢 低 | 纯逻辑层，不依赖系统 API，前端可实现 |
| 摸鱼预警 | ⭐⭐ | 🟡 中 | 窗口标题匹配 + 进程检测，技术路径清晰；边缘 case 需打磨 |
| 表情包生成 | ⭐ | 🟢 低 | Canvas toDataURL + 文字叠加，前端成熟方案 |
| 好友串门 | ⭐⭐⭐⭐ | 🔴 高 | 需 P2P 网络 + 信令服务，非 MVP 范围 |

### 8.2 技术栈推荐

| 方案 | 优点 | 缺点 | 建议 |
| --- | --- | --- | --- |
| **Tauri + Canvas/WebGL** | 内存低（<50MB 基线）、系统 API 调用能力强、安装包小（<10MB） | Rust 学习曲线；插件生态不如 Electron 成熟 | 推荐长期方案 |
| **Electron + Canvas** | 生态成熟、开发快、前端开发者零学习成本 | 内存高（~150MB 基线）、安装包大（>80MB） | 推荐 MVP 快速验证 |

> **建议**：MVP 阶段用 Electron 快速验证产品假设，确认 PMF 后逐步迁移核心模块到 Tauri。项目已有 Node.js 生态配置，Electron 起步更快。

### 8.3 窗口 API 方案对比

| OS | 获取前台窗口 | 获取窗口坐标 | 窗口置顶 |
| --- | --- | --- | --- |
| Windows | `GetForegroundWindow()` (Win32) | `GetWindowRect()` | `WS_EX_TOPMOST` |
| macOS | `NSWorkspace.shared.frontmostApplication` | Accessibility API `kAXPositionAttribute` | `NSWindow.level = .floating` |
| 跨平台封装 | `active-win` (Node) / Tauri `window` API | 同上 | Electron `alwaysOnTop` / Tauri `set_always_on_top` |

## 九、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
| --- | --- | --- | --- |
| 跨窗口坐标适配偏差（不同分辨率/缩放比） | 高 | 中 | 优先适配 1080p/1440p；提供用户手动校准功能（方向键微调宠物位置）；建立社区反馈常见分辨率适配列表 |
| 用户新鲜感过后快速流失（7 天留存问题） | 高 | 高 | 养成系统（长期目标）+ 随机彩蛋（惊喜感）+ 每 2 周更新动作/装扮；效率功能提供实用价值锚点 |
| 冷启动无流量 | 中 | 中 | 录制核心功能视频发布 B站/小红书/GitHub；差异化视觉效果自带传播属性；在程序员社区（V2EX、掘金）精准投放 |
| macOS 权限审核（辅助功能权限） | 中 | 中 | 首次启动提供清晰的权限引导页；无法获取权限时降级为纯桌面悬浮模式 |
| 竞品跟进复制核心功能 | 低 | 高 | 持续更新动作/形象/彩蛋建立内容壁垒；养成数据绑定用户迁移成本；社区规则生态形成网络效应 |
| Windows Defender 误报（全局钩子） | 低 | 中 | 代码签名证书；提交 Microsoft Defender 白名单审核 |
